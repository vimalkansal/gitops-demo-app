üéØ What are DORA Metrics?

DORA (DevOps Research and Assessment) metrics are four key measurements that indicate the performance of a software development team.

üìà Deployment Frequency

How often an organization successfully releases to production

‚è±Ô∏è Lead Time for Changes

Time from code committed to code successfully running in production

üîß Mean Time to Recovery (MTTR)

How long it takes to recover from a failure in production

üìâ Change Failure Rate

Percentage of deployments causing a failure in production

üöÄ Introduction to Pelorus

Pelorus is an open-source tool by Red Hat that helps measure DORA metrics in Kubernetes/OpenShift environments.

Key Components

  • Exporters: Collect metrics from various sources
  • Prometheus: Stores time-series metrics data
  • Grafana: Visualizes metrics in dashboards

Pelorus Exporters

üì¶ Deploytime Exporter

Tracks when deployments occur (Deployment Frequency)

‚è∞ Committime Exporter

Tracks when commits were made (Lead Time)

üî• Failure Exporter

Tracks production failures (MTTR & Change Failure Rate)

üèóÔ∏è Build Exporter

Tracks build information and status

  # Critical for Pelorus metrics collection
  app.kubernetes.io/vcs-ref: "0dfa101..."  # Git commit SHA
  app.kubernetes.io/vcs-uri: "https://github.com/..."
  deploy-time: "2025-08-12T00:32:09Z"
  
  # Build tracking
  app.kubernetes.io/build-id: "WILL_BE_REPLACED_BY_CI"
  app.kubernetes.io/build-url: "WILL_BE_REPLACED_BY_CI"
  
  # Change tracking
  app.kubernetes.io/change-id: "WILL_BE_REPLACED_BY_CI"
  app.kubernetes.io/approved-by: "github-actions"

‚úÖ Implementation Best Practices

üîß CI/CD Pipeline Integration

  • Automate annotation updates in your pipeline
  • Use environment variables for dynamic values
  • Validate annotations before deployment

üìä Metrics Collection

  • Start with one metric at a time
  • Ensure time synchronization across systems
  • Use consistent timezone (UTC recommended)

üè∑Ô∏è Annotation Standards

  • Follow Kubernetes annotation conventions
  • Document your annotation schema
  • Version your annotation format

üîç Monitoring & Alerting

  • Set up alerts for metric degradation
  • Create dashboards for each team
  • Regular metric reviews in retrospectives

‚ö†Ô∏è Common Pitfalls to Avoid

  • ‚ùå Manual annotation updates (automate everything!)
  • ‚ùå Ignoring timezone differences
  • ‚ùå Not validating Git SHA format
  • ‚ùå Missing GitHub token permissions
